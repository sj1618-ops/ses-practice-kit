# SES AI練習キット — インフラ性能問題編

現場で失敗できない人が、安全に失敗できる練習環境です。

## 使い方

1. 各課題フォルダの `README.md` を読む（状況説明だけ書いてある）
2. `logs/` の中のファイルをAIに全部渡す
3. 「何が問題か診断してください」とだけ伝える
4. AIの回答を見て、自分の仮説と比べる
5. `answer.md` で答え合わせ

## AIへの渡し方（例）

```
以下のサーバーログを見て、問題の原因を診断してください。

[vmstat.txtの内容をペースト]
[iostat.txtの内容をペースト]
[top.txtの内容をペースト]
[free.txtの内容をペースト]
```

## 課題一覧

| # | タイトル | 難易度 | 身につくこと |
|---|---------|--------|------------|
| 01 | I/Oウェイト犯人探し | ★☆☆ | topの罠、io_waitの読み方 |
| 02 | シングルスレッドの罠 | ★★☆ | mpstatでコアごとの負荷を見る |
| 03 | ディスク残量あるのに書けない | ★★☆ | inodeとdisk spaceの違い |
| 04 | メモリ余裕あるのに遅い | ★★☆ | swapのsi/soを読む |
| 05 | リソース正常なのに繋がらない | ★★★ | TIME_WAITとポート枯渇 |

## 環境について

ダミーデータはすべて実際のコマンド出力形式に合わせています。  
4コアサーバー（CentOS/RHEL系）を想定したシナリオです。
